## Overview

This is a Hexo blog theme called Apollo, inspired by vuejs.org. It is installed as a theme inside a Hexo blog's `themes/apollo/` directory, not used as a standalone project.

## Build

Compile Tailwind CSS (outputs to `source/css/apollo.css`):

```bash
npm run build   # one-time compile (for production)
npm start       # watch mode — recompiles on pug/CSS changes + starts Hexo on :4000
```

Install dependencies first if needed: `npm install`

The build pipeline is defined in `gulpfile.mjs` (ESM, Gulp 5). Uses PostCSS + Tailwind CSS v4. Requires Node.js ≥ 10.13.

## Architecture

**Templates** use Pug in `layout/`:
- `partial/layout.pug` — base HTML shell; all pages extend this
- `index.pug`, `archive.pug`, `post.pug` — page-level templates that extend `layout`
- `mixins/post.pug` — reusable Pug mixins: `posts()` (home list), `postList()` (archive list), `post(item)` (single post)
- `mixins/paginator.pug` — pagination mixin
- `partial/comment.pug` — Disqus comment integration (conditionally rendered based on `theme.disqus`)

**Styles** use Tailwind CSS v4:
- `source/css/input.css` — entry point: `@import "tailwindcss"`, `@source` for pug scanning, `@layer base` (body/link/code/table defaults, font-face), `@layer components` (tip block, heading `#` markers, code highlighting, paginator, lightbox)
- Tailwind utility classes are applied directly in Pug templates
- Compiled output: `source/css/apollo.css` (generated by `npm run build`)

**i18n** strings live in `languages/en.yml` and `languages/zh-cn.yml`. Accessed in templates via `__('key')`.

## Theme Configuration (`_config.yml`)

Key config options for the theme:
- `menu` — nav links map (label → URL)
- `favicon`, `logo` — asset paths
- `disqus` — Disqus shortname (enables comment widget)
- `startyear` — used in footer copyright

## Post Front-matter

- `desc` — custom meta description (falls back to `title + author`)
- `from` — URL of original post (renders a "Translation · Original Link" label)
- Use `<!--more-->` in post body to set the excerpt shown on the home page

## Tip Block

Posts can render a styled warning block with:
```html
<div class="tip">content</div>
```
